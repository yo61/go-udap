# go-udap v2.3.0 is available

[Get it on the GitHub release page](https://github.com/yo61/go-udap/releases/tag/v2.3.0) — or `brew upgrade go-udap` if you installed via Homebrew.

## What's new since v2.1.0

**Tab completion** for bash, zsh, and fish. Subcommands, flags, the 26 NVRAM parameter names — all complete. Two highlights:

- Tab after `go-udap info ` runs a quick discovery and offers MAC addresses of devices currently on your network. No more typing six hex bytes with colons.
- Tab after `--bind-interface ` offers your usable local interface names.

If you installed via Homebrew (`brew install yo61/tap/go-udap`), completions wire themselves up on the next `brew upgrade` — no manual setup.

**SBOMs attached to every release.** Each release archive on GitHub now ships alongside an SPDX and a CycloneDX bill-of-materials file (`.spdx.json` / `.cdx.json`). If your environment requires SBOMs for review, you can grab them next to the binary. They're covered by `SHA256SUMS` so you can verify integrity.

**Daily vulnerability monitoring.** A scheduled scan now checks go-udap's dependencies against CVE databases every day. If a vulnerability is disclosed against something we depend on, it gets surfaced within 24 hours instead of waiting for the next dev cycle.

**Things you might notice in passing:**

- The default network timeout dropped from 5s to 2s.
- Help and error message formatting changed slightly — the CLI moved to the standard Cobra framework under the hood. This is what enables the completions above and gives more polished diagnostics. Existing commands, flags, and exit codes are unchanged.
- On the [docs site](https://yo61.github.io/go-udap/), in-site search and social-preview images now render correctly.
