# Homebrew tap for go-udap

## Goal

Make `go-udap` installable via Homebrew on macOS and Linux, with the formula auto-published on every tagged release. Write the formula to homebrew-core audit standards from day one so eventual migration to homebrew-core is a copy-paste rather than a rewrite.

## Out of scope

- A Homebrew **Cask** — `go-udap` is a CLI binary, not a `.app`/GUI. Formula is correct.
- **Bottle hosting** — Homebrew will build from source from the GitHub release tarball; bottles aren't worth the CDN infra for a small Go binary.
- **Submitting to homebrew-core now** — homebrew-core has notability criteria (stars, age, "notable") that this niche Squeezebox tool doesn't currently meet. Tracked as a future migration; see [Path to homebrew-core](#path-to-homebrew-core) below.
- **Auto-deprecating older formula versions** — the tap keeps a single formula at the latest tag.

## Architecture

A new public repo **`yo61/homebrew-tap`** hosts the formula. The existing `goreleaser.yaml` workflow gains a `brews:` block that, on every tag matching the existing SemVer regex, generates `Formula/go-udap.rb` and pushes it to the tap.

End-user install:

```sh
brew install yo61/tap/go-udap
```

(Homebrew automatically resolves `yo61/tap` to `https://github.com/yo61/homebrew-tap`; the `homebrew-` prefix is convention.)

## Components

### 1. The tap repo (`yo61/homebrew-tap`)

```
homebrew-tap/
├── Formula/
│   └── go-udap.rb              # generated by goreleaser, never hand-edited
├── .github/
│   └── workflows/
│       └── audit.yaml          # brew audit + brew test on PR/push
├── README.md                   # install instructions
└── LICENSE                     # MIT, matching go-udap
```

The `audit.yaml` workflow:

- Triggers on `pull_request` and `push: branches: [main]`.
- Uses `Homebrew/actions/setup-homebrew@master` to get a working `brew`.
- Runs `brew audit --strict --new Formula/go-udap.rb` (the same audit homebrew-core PRs face).
- Runs `brew install --verbose Formula/go-udap.rb` then `brew test go-udap` against the just-installed copy.
- Declares `permissions: contents: read` at workflow scope (project standard).

### 2. goreleaser config addition

Append a `brews:` block to `.goreleaser.yaml`:

```yaml
brews:
  - name: go-udap
    repository:
      owner: yo61
      name: homebrew-tap
      branch: main
      token: "{{ .Env.HOMEBREW_TAP_GITHUB_TOKEN }}"
    homepage: "https://github.com/yo61/go-udap"
    description: "Squeezebox UDAP configuration tool"
    license: MIT
    test: |
      assert_match "go-udap", shell_output("#{bin}/go-udap --version")
      assert_match "Usage:", shell_output("#{bin}/go-udap --help")
      # exit-1 path: invalid subcommand should produce a usage error
      assert_match "unknown command", shell_output("#{bin}/go-udap floop 2>&1", 1)
    install: |
      bin.install "go-udap"
```

`name`, `homepage`, `description`, `license`, and `test do` are all required by `brew audit --strict --new`.

The test exercises three things:

- The version flag (a stdout assertion on the binary at all).
- The help flag (the CLI dispatcher's happy path).
- An invalid-subcommand path (asserts exit code 1 — the project's documented usage-error code).

No network is touched in any of the tests.

### 3. goreleaser workflow auth

The current `goreleaser.yaml` workflow uses the default `GITHUB_TOKEN`, which is scoped to `yo61/go-udap` only. To push commits to `yo61/homebrew-tap`, the workflow gains an App-token step mirroring the pattern in `release.yaml`:

```yaml
permissions:
  contents: write   # for `gh release edit --draft=false` on this repo

jobs:
  goreleaser:
    runs-on: ubuntu-latest
    steps:
      - id: tap_token
        uses: actions/create-github-app-token@v3
        with:
          client-id: ${{ secrets.SEMANTIC_RELEASE_APP_CLIENT_ID }}
          private-key: ${{ secrets.SEMANTIC_RELEASE_APP_PRIVATE_KEY }}
          owner: ${{ github.repository_owner }}
          repositories: |
            homebrew-tap

      - uses: actions/checkout@v6
        with:
          fetch-depth: 0

      # ...setup-go, etc...

      - uses: goreleaser/goreleaser-action@v7
        with:
          distribution: goreleaser
          version: '~> v2'
          args: release --clean
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          HOMEBREW_TAP_GITHUB_TOKEN: ${{ steps.tap_token.outputs.token }}
```

`GITHUB_TOKEN` (scoped to this repo) continues to drive the GitHub release upload. `HOMEBREW_TAP_GITHUB_TOKEN` is consumed only by goreleaser's `brews:` block to push the formula to the tap repo.

### 4. GitHub App installation extension

One-time, on GitHub:

- The existing `yo61`-org GitHub App (the one minting `SEMANTIC_RELEASE_APP_CLIENT_ID`) needs its **installation** extended to also include the new `yo61/homebrew-tap` repo.
- Org settings → GitHub Apps → the App → Configure → "Repository access" → add `homebrew-tap`.

No new secrets. `SEMANTIC_RELEASE_APP_CLIENT_ID` and `SEMANTIC_RELEASE_APP_PRIVATE_KEY` already in place are reused.

## Data flow

```
git tag vX.Y.Z
    │
    ▼
push tag to yo61/go-udap
    │
    ▼
.github/workflows/goreleaser.yaml fires
    │
    ├─► mint installation token for [go-udap, homebrew-tap]
    ├─► checkout, setup-go
    ├─► goreleaser release --clean
    │       │
    │       ├─► build cross-platform binaries
    │       ├─► create draft GitHub release (yo61/go-udap), upload tarballs + SHA256SUMS
    │       └─► generate Formula/go-udap.rb, push to yo61/homebrew-tap@main
    └─► gh release edit --draft=false   (publishes the go-udap release)

end-user later:
    brew install yo61/tap/go-udap
        │
        ├─► brew clones yo61/homebrew-tap
        ├─► reads Formula/go-udap.rb
        ├─► fetches the platform-matching tarball + verifies SHA256
        └─► installs go-udap into the Cellar
```

## Error handling

| Failure | Effect | Mitigation |
|---|---|---|
| App not installed on `homebrew-tap` | `actions/create-github-app-token` step 404s | Documented one-time setup step; tap workflow fails loudly on first release if missed. |
| `brew audit` fails on the generated formula | Bad formula gets pushed to the tap; user installs break | Tap repo's `audit.yaml` runs on every push to `main`, so a broken formula surfaces as a failed CI run within minutes of release. Doesn't *prevent* the bad push, but flags it for hand-fix or revert. |
| Tarball SHA256 doesn't match what `Formula/go-udap.rb` claims | `brew install` fails with checksum error | goreleaser computes the SHA from the same tarball it uploads, so this can't happen unless the GitHub release is tampered with after the fact. |
| Tag created without a corresponding release | `brews:` step has no artifacts to reference | Already prevented by the existing tag pattern guard in `goreleaser.yaml`. |

## Testing

**At release time** (`goreleaser.yaml`): goreleaser's own `brews:` step generates the formula and pushes it. No formula validation here — that's the tap repo's job.

**On the tap repo** (`audit.yaml`):

- `brew audit --strict --new Formula/go-udap.rb` — same audit a homebrew-core PR would receive. Catches lint, naming, license, test-block, and dependency issues.
- `brew install --verbose Formula/go-udap.rb` followed by `brew test go-udap` — verifies the formula's `install` block places the binary correctly and the `test` block passes.
- Runs on Linux runner only for v1; macOS runner can be added later if formula complexity grows. The audit and test logic is platform-independent for a Go binary.

## Path to homebrew-core

When `go-udap` accumulates enough notability for homebrew-core (typical bar: ~75+ stars, ≥30 days old, "notable"):

1. Run `brew audit --strict --new --online --git Formula/go-udap.rb` against the tap formula. The `--online` and `--git` flags add the extra checks core enforces (URL reachability, git history sanity). Should already pass given the audit-on-every-push regime.
2. Fork `homebrew/homebrew-core` and open a PR with the formula adjusted:
   - Drop the `repository:` block (irrelevant to core).
   - The `url` and `sha256` already point at the GitHub release tarball — those carry over unchanged.
3. Once merged in core, mark the tap formula as superseded:
   - Either delete `Formula/go-udap.rb` from the tap (simplest) and update the tap README to point users at `brew install go-udap`.
   - Or keep a `disable!` stub in place for one cycle so existing tap users get a clear "moved to homebrew-core" message on next `brew upgrade`.

No design changes are needed today — the v1 formula is already written to core's bar.

## Implementation outline

The implementation plan (separate doc) will cover, in order:

1. Create `yo61/homebrew-tap` repo on GitHub with README, LICENSE, empty `Formula/` dir, and `.github/workflows/audit.yaml`.
2. Extend the GitHub App installation to include `homebrew-tap`.
3. Extend `.goreleaser.yaml` with the `brews:` block.
4. Extend `.github/workflows/goreleaser.yaml` with the App-token step and env var.
5. Cut a test release to verify end-to-end (could be a `vX.Y.Z-rc.1` prerelease so it doesn't pollute the public version sequence).
6. Update `README.md` install instructions to mention the tap alongside the existing manual download.
